The Cascade

CSS is an acronym of Cascading Style Sheets,which indicates that the notion of the cascade is important.At its most basic level indicates that the order of CSS rules matter,but it's more complex than that.What selectors win out in the cascade depends on three factors(these are listed in order of weight earlier ones will overrule later ones):

    1.Importance

    In CSS,there is a special piece of syntax you can use to make sure that a certain declaration will always win over all others:!important.

    2.Specificity

    Specificity is basically a measure of how specific a selector-how many elements it could match.Class selector have an even higher specificity,so will win against element selector.ID selectors have an even higher specificiy,so will win against class selectors.The only way to win against an ID selector is to use!important.

    Selector Thousands Hundreds Tens Ones Total Specificity
    h1 0 0 0 1 0001
    #important 0 1 0 0 0100
    h1 + p::first-letter 0 0 0 3 0003
    li > a[href*="en-US"] > .inline-warning 0 0 2 2 0022
    #important div > div > a:hover,inside an element's style attribute 1 1 1 3 1113

    NOTE:If multiple selector have the same importance and specificity,which selector wins is decided by which comes later in the Source Order

    3.Source Order

    As mentioned above,if multiple competing selectors have the same importance and specifity,the third factor that comes into play to help decide which rule wins is source order---later rules will win over earlier rules.
